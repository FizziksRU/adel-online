# А.Д.Е.Л.Ь. — онлайн-стол

Неофициальная цифровая реализация настольной игры «А.Д.Е.Л.Ь.» (изд. Magellan) для игры с друзьями по сети. Механики закодированы по буклету правил; **арты и тексты оригинальных карт не используются**.

> ⚠️ Это фанатский стол для личных партий. Не выкладывайте его в открытый доступ с оригинальными материалами игры и не используйте коммерчески.

## Запуск локально

```bash
npm install
npm run build     # собрать клиент
npm start         # сервер на http://localhost:8000
```

Для разработки: `npm run dev` (Vite на :5173 + сервер на :8000).

> ⚠️ Клиент подхватывает правки через hot-reload, а **сервер — нет**: он держит
> загруженный движок в памяти. После изменений в `src/game/` перезапускайте
> `npm run dev`, иначе клиент будет рисовать новый код поверх старого состояния
> партии (симптом — пустой экран и ошибка в `<Board>`).

## Как играть

1. Откройте сайт и выберите состав: **3 игрока** (АДЕЛЬ + 2 члена экипажа, 18 ходов) или **4 игрока** (АДЕЛЬ + 3, 15 ходов). Нажмите «Создать партию».
2. Отправьте друзьям появившуюся ссылку — каждый выбирает себе место (АДЕЛЬ — место 0).
3. Ход идёт по фазам: событие → планирование (все тайно) → фаза АДЕЛЬ → розыгрыш → действия экипажа → конец хода. Интерфейс сам показывает нужную панель. Отдельное окно между событием и планированием появляется, только если АДЕЛЬ активировала аномалию «Атака».
4. **Проверка духа — ваш ход, а не бросок сервера.** Когда очередь дошла до вас, поверх экрана появляется плашка с причиной и порогом, и до нажатия «Бросить кубик» игра стоит для всех. Результат с анимацией видят все за столом, включая АДЕЛЬ и зрителей.
5. Консоль АДЕЛЬ лежит под картой корабля и открыта всем — как планшет на настоящем столе. Шкалы ран тоже открыты: здоровье в этой игре публично.
6. Общайтесь голосом (Discord/Telegram) — по правилам АДЕЛЬ слышит всё, что говорится вслух. Тайный обмен — только кнопками «показать/передать» при встрече в одной локации.

## Деплой

### Railway (проще всего)
1. Залейте папку в GitHub-репозиторий.
2. railway.app → New Project → Deploy from GitHub. Railway сам увидит Dockerfile.
3. В Settings → Networking включите публичный домен. Готово.

### Fly.io
```bash
fly launch --no-deploy   # согласитесь с Dockerfile, внутренний порт 8000
fly deploy
```

Партии переживают перезапуск сервера: состояние лежит в файлах (`FlatFile`), каталог задаётся `DB_DIR` (в образе — `/data`, объявлен томом). При развёртывании подключите к `/data` постоянный том, иначе партии потеряются при пересоздании контейнера.

### Переменные окружения

| Переменная | Зачем | По умолчанию |
| --- | --- | --- |
| `PORT` | порт сервера | `8000` |
| `DB_DIR` | каталог с партиями | `./.matches` (в образе `/data`) |
| `ALLOWED_ORIGIN` | откуда принимать подключения, через запятую | только localhost |

Без `ALLOWED_ORIGIN` сервер принимает только локальные подключения — при деплое укажите там адрес своего домена.

Доступ «по коду комнаты»: ссылка на партию содержит непубличный matchID — этого достаточно для игры с друзьями. Публичного списка партий в интерфейсе нет.

## Данные коробки

Состав компонентов сверен с физической коробкой и собран в [COMPONENTS.md](COMPONENTS.md) — предметы, фишки, консоль АДЕЛЬ, события, аномалии, поле, персонажи. Все числа оттуда закодированы в `src/game/data.js` и проверяются тестами: суммы (32 жетона предметов, 52 фишки, 52 ячейки консоли, 25 карт событий, 27 карт АДЕЛЬ) должны сходиться, иначе `npm test` упадёт.

Сверка завершена: несверенных пунктов не осталось.

Поддержаны два состава: **АДЕЛЬ + 2** (3 игрока, 18 ходов) и **АДЕЛЬ + 3** (4 игрока, 15 ходов). Состав на 5 игроков не открыт: не разобран четвёртый планшет персонажа и не читаются строки таблицы маркеров для 4–5 членов экипажа (`MARKER_TABLE` в `data.js`).

## Упрощения цифровой версии

- Кубики бросает сервер, но **проверку духа запускает игрок**: она ставится в очередь и ждёт кнопки, а результат разыгрывается на сервере и анимируется у всех. Остальные броски (d20 события, заряд предметов) идут молча, результат — в журнале.
- Терминал доставки «забрать предмет» требует согласия владельца — подтверждайте голосом, кнопка согласие не спрашивает.
- Доплату кубиком за «тьму» и «вредоносную программу» движок берёт сам, с первого действия, где кубик есть; по правилам его выбирает игрок.
- Все семь специальных карт АДЕЛЬ замешаны в общую колоду; правило подготовки «уберите две для первой партии» отключено константой `SPECIALS_REMOVED`.

## Структура

```
src/game/data.js    — все игровые данные (правьте здесь)
src/game/index.js   — движок правил (boardgame.io)
src/client/         — React-интерфейс
  Board.jsx          — экран партии: карта корабля, панели фаз, сайдбар
  Console.jsx        — консоль АДЕЛЬ под картой (раскладка физического планшета)
  Health.jsx         — шкалы ран экипажа (публичные)
  SpiritRoll.jsx     — плашка проверки духа и анимация кубика
  AdelCards.jsx      — панель выбранной карты АДЕЛЬ
server.js           — сервер (boardgame.io + статика)
test/                — тесты (npm test), 748 проверок:
  smoke.js           — полный цикл хода через настоящий boardgame.io, оба состава
  rules.js           — точечные правила и состав коробки
  crew.js            — правила экипажа: тьма, огнетушитель, перегруз, шпионаж
  specials.js        — семь специальных карт АДЕЛЬ
  spirit.js          — очередь проверок духа и рискованное спец. действие
  endings.js         — четыре концовки по отдельности
  playthrough.js     — пять полных отыгранных партий
  bot.js             — бот экипажа для них: ищет предметы, встречается, доставляет
  ui.js              — рендер настоящих компонентов интерфейса
  layout.js          — раскладка карты корабля
```
